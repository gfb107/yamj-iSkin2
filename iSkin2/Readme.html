iSkin2: The YAMJ skin for Tablets, Laptops, and Desktops

iSkin2 brings YAMJ to any Tablet, Laptop, or Desktop computer with a web-kit based browser. This includes iPads, Android Tablets, or any Laptop or Desktop with Chrome or Safari.

Browse or search your video collection on your device, then once you've decided what you want to watch, start watching it with one final touch, and then control playback using the integrated remote.

Information

iSkin2 is an HTML skin for YAMJ. It is unique among YAMJ skins in that it is not viewed directly on the NMT. It is viewed on an internet connected device, such as an iPad, Android tablet, a laptop or desktop computer.

In order for this to work, the generated HTML files must be accessible by an HTTP URL from an HTTP server on your home network. An NMT with an internal HDD can be used as the HTTP Server (install the NMT Applications and make sure myiHome is running), but WHS (Windows Home Server), and most file servers and NAS (Network Attached Storage) devices include one or have one or more that can be installed.

If you are currently using a YAMJ skin with your NMT, you can use iSkin2 in conjunction with that skin, but you must run YAMJ twice. Once for the "normal" skin that is viewed on the TV connected to the NMT, and a second time for iSkin2.

Requirements:

  1. An internet enabled device with a web-kit based browser (Apple iPad, Android tablet; laptop or desktop computer running Google Chrome or Apple Safari) with access to your home network.
  2. YAMJ. Through out the rest of this document, I'll assume it's installed in C:\YAMJ. If you've installed it somewhere else, you'll have to substitute the actual installation directory where ever you see C:\YAMJ
  3. A Network Media Tank (NMT)
    • If you have a Series 1 NMT, it must have:
      • An internal HDD (hard disk drive) or other storage on which the NMT Applications can be installed,
      • NMT Applications installed with FTP Server running, as required by the NMT Community Software Installer,
      • PLoNK-a100-csi.0.0.3.zip or newer.

        Use NMT Community Software Installer to install the appropriate package on each of your NMTs. From the File menu, select "Install from file..." and follow the instructions.

  4. An HTTP Server to make the HTML files available for browsing. You have a number of options here:
    • If you have an internal HDD in your NMT, you can use the myiHome NMT Application. The document root for the HTTP Server is the root of the internal HDD, and the base URL to the document root will be http://ip.address.of.nmt:8088/stream/file=/share/
    • If you have a Windows Home Server (WHS), you can use Internet Information Server (IIS). The document root for IIS is C:\Inetpub\wwwroot, and the URL to the document root is http://ip.address.of.whs/
    • Most NASes include an HTTP server that you can use. If not, apache, lighttpd or llink are often available. You'll need to figure out what the document root is, as well as the URL to the document root.
  5. File system access to the document root folder of your HTTP Server, often achieved with an SMB (or NFS) mount. In the rest of this document, I'll refer to this folder as H:.
    • If you are using your NMT's internal drive, mount it as H:
    • If you are using a WHS, you can run YAMJ on the WHS, and the document root folder will be C:\Inetpub\wwwroot\, so use that instead of H:
    • If you are using another file server or NAS, you'll have to figure out what the document root is, and how to access it on your PC. Whatever that path is, use that instead of H: in the following instructions. You'll also need to know the url to the document root.

Instructions:

These instructions assume you've already satisfied the above requirements and verified that each one is operating correctly. Please do not proceed any further unless you have done that.

  1. Unzip iSkin2-preview6a.zip into the skins directory of your YAMJ installation ( C:\YAMJ\skins ). This will create the iSkin2 directory, next to the other skin directories, such as default, gfb107, gfb107-sd, and sd.
  2. In the newly created iSkin2 folder, edit skin-options.xsl and
    1. Set skin-nmtList as follows:

      Create a comma separated list of entries for each NMT you have. Each entry consists of:

      • The name of the NMT, or the room in which the NMT resides
      • A colon
      • The ip address of the NMT
      • A colon
      • The series/generation of your NMT (1, 2, 3, or 4). For example, an A-110 is a series 1, a C-200 is a series 2, an A-300 is a series 3, an A-400

      Here is an example for one room, with a series 2 NMT:

      <xsl:variable name="skin-nmtList">Family Room:192.168.1.24:2</xsl:variable>

      Here is an example adding a second room with a Series 1 NMT:

      <xsl:variable name="skin-nmtList">Family Room:192.168.1.24:2,Bonus Room:192.168.1.25:1</xsl:variable>

      Note: Using a single instance of iSkin2 with multiple NMTs requires careful configuration:

      • Every NMT must be able to access your entire video collection using the same NMT paths.
      • Every Series 1 NMT must meet the requirements explained above
    2. If you are using a Series 1 NMT, set skin-doneURL to the URL you want displayed on your TV by the NMT when a video stops playing. The default value of file:///share/index.htm works for those with an HDTV and are using a custom index (or one generated by YAMJ). Those with SDTVs should use file:///share/index_sd.htm.

      If your videos are accessed through a network share, your custom index should also be accessed through that network share, and your skin-doneURL should be the NMT path to the index on that share, something like file:///opt/sybhttpd/localhost.drives/NETWORK_SHARE/MOVIES/index.htm. Note how similar this value is to the NMT path you entered into moviejukebox.properties or library.xml when setting up YAMJ.

      Note: the skin-doneURL is only used with Series 1 NMTs. Series 2 and 3 NMTs will always return to the screen that was displayed before the video started playing.

  3. Use command-line option to run YAMJ using iSkin2 skin and to put the generated HTML file in the HTTP Server's document root:
    MovieJukebox library.xml -D mjb.skin.dir=./skins/iSkin2 -o H:
    Substitute the path to the document root of your HTTP Server instead of H:, if needed.
  4. Open the browser on your internet connect device at point it at iSkin.html on your HTTP Server. Here are some examples:
    • If you are using your NMTs internal HDD and the myiHome NMT Application:
      http://ip.address.of.nmt:8088/stream/file=/share/iSkin2.html
    • If you are using a WHS or NAS device
      http://ip.address.of.device/iSkin2.html
  5. Put an icon for iSkin2 on your Home Screen. iPad/iPhone users must NOT skip this step, or else scrolling will not work correctly.
    1. Touch the "+" in the status bar
    2. Touch "Add To Home Screen"
    3. Touch "Add"
    4. Touch the new icon for iSkin2 on your Home Screen to go directly to iSkin2 in full screen mode
